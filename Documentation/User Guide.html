Login Page


The username and password you use to get into Grant Manager are stored in the database. If your Grant Manager user account is marked as an administrator, you will be able to add and remove accounts from the User Management screen. If you set up the database using the instructions in Instructions.txt, the default user account has the username "admin" and the password "admin".

The About button shows the open-source licenses used by the program and has a link to the code on GitHub.

Main Page

The main page appears when you first log into Grant Manager. It gives you access to the Grants, Grantors, and Projects screens, as well as the Administration menu.

Two types of alerts appear in the calendar: grant due dates, which always appear in red, and grant timeline dates, which can have custom names and colors. Clicking a day on the calendar will show that day's alerts (due dates and timeline dates) in the "selected day's alerts" pane. The "upcoming alerts" page will always show all alerts (due dates and timeline dates) within the next month.

If you double-click an alert in either "selected day's alerts" or "upcoming alerts", you will be taken to the Edit Grant or Timeline Editor screen for the grant with which that alert is associated.

Edit Text

Several text fields in Grant Manager have a small "Ex" button next to them. When you click this button, it will open a new window, which will be pre-populated with the existing text (if any).
When you close this new window by pressing OK, the text you entered will be put back into the appropriate field in the original window.

Your text will not actually be saved to the database until you click the Submit button on the original window (e.g. Grant Editor; Grantor Editor.)

Grants


The Grants screen lists all of the grants in your system. You can adjust the column headers to change the width of the columns (similar to Microsoft Excel.)
You can also use the search bar: type part of the grant name into the bar and press the Search button (or Enter on your keyboard.) The "X" button will clear the search bar and show all grants.

To edit a grant, click on its row in the list and then click the Edit button.

Besides their basic properties (shown on the Grant Editor), grants have three sets of additional properties you can edit:
  • Budget items (click the Budget button)
  • Custom timeline dates (click the Timeline button)
  • Contact history (click the Edit Contact History button)

In order to edit a grant's contact history, you will need to define at least one approach type (such as "email" or "in person") in Administration Approaches.

Grant Editor

The Grant Editor lets you edit the basic properties of the grant, upload/download attachments, and associate it with any number of projects and/or programs. To put anything in Projects, Programs or Requirements, you must first add possible programs or requirement types to Grant Manager by using the Administration window.

Five types of dates are defined: submit date, due date, turnaround date, start date, and payment date. All five are optional, and only the due date will appear in the calendar on the main page.

You must click "Submit" to save your changes to the grant.

Program Emphasis and Notes are text fields. If you want to edit their text in a bigger window, click the small "Ex" button in the upper-right corner of the box to expand them into a new window (see the "Edit Text" section near the top of this guide.)

Attachments

To edit an attachment, select it in the list on the Grant Editor and click the Edit button below the list. (You can also add a new attachment by using the Add button right next to it.)

Any number of files can be attached to a grant. You cannot attach folders, but you can attach any type of file, including zip files.

The "upload" button will replace the existing attachment.
The "download" button lets you save the attachment to your computer.
The "view copy" button saves the attachment to a hidden temporary folder on your computer and opens it for you.
The "delete" button deletes the attachment from the Grant Manager database.

Budget Editor


This screen stores budget items for a particular grant, and can be accessed from the Budget button on the Grants screen. Revenue items can be entered in the upper list, and expenses can be entered in the lower list. The screen also lists total revenue, total expenses, and a grand total (which can be positive or negative.)

Timeline Editor


This screen stores custom dates for a particular grant, and can be accessed from the Timeline button on the Grants screen. All dates entered here will appear on the calendar on the main page with the selected color.

When you open the Timeline Editor, the dates for the selected grant will be listed in chronological order.

Contact History

The contact history is an additional set of information associated with each grant. The options in the "initial approach" drop-down list can be configured in Administration Approaches.

The "Ex" buttons next to the Outcome and Notes fields open those fields in a new window. See the "Edit Text" section near the top of this guide for an explanation.

Grantors


The Grantors screen lists all of the grantors in your system. You can adjust the column headers to change the width of the columns (similar to Microsoft Excel.)
You can also use the search bar: type part of the grantor name into the bar and press the Search button (or Enter on your keyboard.) The "X" button will clear the search bar and show all grantors.

To edit a grantor, click on its row in the list and then click the Edit button.

Grantor Editor


The Grantor Editor lists properties of the grantor and allows you to edit them. You must click "Submit" to save your changes to the grantor.

Type of Support, Giving History, Buzzwords, and Notes are text fields. If you want to edit their text in a bigger window, click the small "Ex" button in the upper-right corner of the box to expand them into a new window (see the "Edit Text" section near the top of this guide.)

Projects

The Projects screen lists all of the projects in your system. (Projects can be associated with any number of grants.) You can adjust the column headers to change the width of the columns (similar to Microsoft Excel.)
You can also use the search bar: type part of the project name into the bar and press the Search button (or Enter on your keyboard.) The "X" button will clear the search bar and show all projects.

To edit a project, click on its row in the list and then click the Edit button.

Project Editor

The Projects Editor lists properties of the project and allows you to edit them. You must click "Submit" to save your changes.

All fields except Name are multi-line text fields. If you want to edit their text in a bigger window, click the small "Ex" button in the upper-right corner of the box to expand them into a new window (see the "Edit Text" section near the top of this guide.)

Administration


The Administration screen is accessible from the main page.

Administrators will get a User Management option, while other users will get Change Password. Both are described below.

Programs, Requirement Types, and Approaches bring up screens similar to the Grantors and Projects screens. Programs and requirement types can be attached to grants (see the "Grant Editor" section); approaches can be selected for a grant's contact history (see "Contact History.")

"Export Change Log" will export a detailed changelog stored in the Grant Manager database. This log records all changes to grants, grantors, projects, contact history, budget items, and grant timeline dates within the last month. (Entries older than 30 days are deleted from the database automatically to save space.) The exported file will have a .csv extension, and can be opened by Microsoft Excel or any text editor.

User Management / User Editor


The User Management screen is only accessible to administrators. It lists all users in the system, whether they can log in ("Active"), and whether they are an administrator.

If a user is listed as a "grant writer" on any grant or under "contacted by" on any grant's contact history, they cannot be deleted from the database, but you can prevent them from logging in by setting their user record to be inactive using the User Editor.

Administrators can also change other user's passwords or give/remove admistrator permission to other users. They cannot remove their own administrator permission.

Change Password


Non-administrators will be able to access this screen, which allows them to change their own password or display name. They cannot make themselves an administrator or access any other user accounts.
